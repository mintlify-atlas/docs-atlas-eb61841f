---
title: "Quickstart"
description: "Get Squiggle running locally in under 10 minutes."
---

<Steps>
  <Step title="Clone the repo and install dependencies">
    Clone the repository and install dependencies with your preferred package manager.

    ```bash
    git clone https://github.com/thor-op/squiggle.git
    cd squiggle
    ```

    <CodeGroup>

    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```

    </CodeGroup>
  </Step>

  <Step title="Set up Firebase">
    Squiggle uses Firebase Firestore for real-time room state, canvas sync, and chat. You need a Firebase project with Firestore enabled before you can run the app.

    1. Create a project at [console.firebase.google.com](https://console.firebase.google.com)
    2. Enable **Firestore** in Native mode
    3. Go to **Project Settings → Service Accounts → Generate new private key** to download your Admin SDK credentials

    See [Firebase Setup](/setup/firebase) for a full walkthrough including Firestore index configuration and security rules.
  </Step>

  <Step title="Configure environment variables">
    Copy the example file and fill in your Firebase credentials.

    ```bash
    cp .env.example .env.local
    ```

    Open `.env.local` and set the following variables:

    | Variable | Description |
    |---|---|
    | `NEXT_PUBLIC_FIREBASE_API_KEY` | Firebase web API key |
    | `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN` | e.g. `your-project.firebaseapp.com` |
    | `NEXT_PUBLIC_FIREBASE_DATABASE_URL` | Realtime Database URL (required by the Firebase SDK) |
    | `NEXT_PUBLIC_FIREBASE_PROJECT_ID` | Your Firebase project ID |
    | `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET` | e.g. `your-project.firebasestorage.app` |
    | `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID` | Sender ID from the Firebase console |
    | `NEXT_PUBLIC_FIREBASE_APP_ID` | Firebase app ID |
    | `NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID` | Analytics measurement ID (e.g. `G-XXXXXXXXXX`) |
    | `NEXT_PUBLIC_FIREBASE_FIRESTORE_DB` | Custom Firestore database ID — leave blank to use `(default)` |
    | `FIREBASE_ADMIN_PROJECT_ID` | Same as your client project ID |
    | `FIREBASE_ADMIN_CLIENT_EMAIL` | Service account email from the downloaded key file |
    | `FIREBASE_ADMIN_PRIVATE_KEY` | Private key from the downloaded key file (include the full `-----BEGIN PRIVATE KEY-----` block) |

    <Warning>
      Variables prefixed with `NEXT_PUBLIC_` are exposed to the browser. Never add your Admin SDK credentials (`FIREBASE_ADMIN_*`) as `NEXT_PUBLIC_` variables — they are server-side only and must remain secret.
    </Warning>

    See [Environment Variables](/setup/environment) for a full reference on each variable.
  </Step>

  <Step title="Deploy Firestore security rules">
    Squiggle ships with Firestore rules that block client reads on the `wordBank` collection so players cannot read words from the browser. Deploy the rules before running the app.

    ```bash
    npx firebase login
    npx firebase use your-project-id
    npx firebase deploy --only firestore:rules
    ```

    Replace `your-project-id` with the value you set for `NEXT_PUBLIC_FIREBASE_PROJECT_ID`.
  </Step>

  <Step title="Seed the word bank">
    Words are stored in Firestore under `wordBank/{category}` and fetched server-side only via the Admin SDK. The client bundle contains no words — only a `getWordMask` utility that converts a word into underscores for the hint display.

    <Note>
      The word bank is server-side only. Firestore security rules block all client reads on the `wordBank` collection, so players cannot inspect the source or network traffic to find the secret word. Your word list lives in `scripts/seed-words.ts`, which is gitignored — only the example template is committed to the repository.
    </Note>

    Copy the template, fill in your categories and words, then run the seed script:

    ```bash
    cp scripts/seed-words.example.ts scripts/seed-words.ts
    # edit scripts/seed-words.ts with your categories and words
    npm run seed
    ```

    The `seed` script runs `scripts/seed-words.ts` via `tsx` using your Admin SDK credentials from `.env.local`. To update the word bank later, edit the file and re-run `npm run seed`.

    See [Word Bank](/setup/word-bank) for the full schema, category structure, and guidance on adding custom words.
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    Open [http://localhost:3000](http://localhost:3000). You should see the Squiggle landing page where you can enter a name and create a private room.

    Room codes are 6 characters (e.g. `A3K9BZ`), generated randomly from uppercase letters and digits. Share the code with friends to let them join. The host controls rounds, draw time, hints, word categories, and max players from the lobby before starting the game.
  </Step>
</Steps>
